WebSocket 协议

2.1 微信扫码登录 WebSocket RocketMQ

1
前端建立 WebSocket 连接
ws://host:8091/ → Netty 握手 → HttpHeadersHandler 提取 token/IP → WebSocketServerProtocolHandler 升级
2
前端请求登录二维码
发送 {"type":1} → WebSocketService.handleLoginReq(channel) → 生成 loginCode → WxMpService.qrCodeCreateTmpTicket() → 返回 {"type":1, "loginUrl"}
3
用户微信扫码
微信发送 SCAN/SUBSCRIBE 事件到 WxPortalController.post() → WxMpMessageRouter → ScanHandler.handle()
4
处理扫码事件
WxMsgService.scan() — 查/建 User → Redis 存 openid:{openId}→loginCode → 回复微信授权链接
5
广播扫码成功
ScanSuccessMessageDTO → RocketMQ user_scan_send_msg (BROADCAST) → ScanSuccessConsumer → WebSocketService.scanSuccess() → 前端收到 {"type":2}
6
用户授权
点击授权链接 → OAuth2 回调 WxPortalController.callBack(?code=) → 换 accessToken → 获取 WxOAuth2UserInfo
7
授权处理
WxMsgService.authorize(userInfo) — 更新用户信息 → Redis 取 loginCode → 发 LoginMessageDTO
8
广播登录完成
MsgLoginConsumer → WebSocketService.scanLoginSuccess() → LoginService.login(uid) 签发 JWT (5天) → 存 Redis userToken:uid_{uid} → 前端收到 {"type":3, "uid/avatar/token/name"}

2.4 发送消息全链路 REST RocketMQ WebSocket

1
HTTP 鉴权
TokenInterceptor.preHandle() — 提取 Authorization: Bearer <token> → LoginService.getValidUid(token) 查 Redis → 设置 request.uid
2
Controller 频控
POST /api/v1/channels/{id}/messages → MessageController.sendMsg() — @FrequencyControl(5s×3, 30s×5, 60s×10)
3
Service 权限检查
MessageService.sendMsg() → PermissionService.checkPermission(serverId, uid, channelId, SEND_MESSAGES) — 成员身份 + RBAC 权限 + Thread-only 校验
4
策略选择 + 处理器执行
MsgHandlerFactory.getStrategyNoNull(msgType) → AbstractMsgHandler.checkAndSaveMsg() — JSR303 → checkMsg() → saveMsg() — 含敏感词 AC 自动机过滤、URL 提取、@提及存储
5
持久化 + 发布事件
MessageAdapter.buildMsgSave() → messageDao.save() → publishEvent(ChannelMessageSendEvent)
6
事务提交后投递 MQ
MessageSendListener.messageRoute() @TransactionalEventListener(AFTER_COMMIT) → MQProducer.sendSecureMsg() @SecureInvoke → RocketMQ MSG_SEND_TOPIC
7
MQ 消费 + 构建 VO
MsgSendConsumer — 从 DB 加载 Message → MessageAdapter.buildChannelMessageVO() — 含 fromUser / reactions / attachments
8
路由推送
threadId≠null → 推给 Thread 订阅者 | threadId=null → 推给 Channel 订阅者 | 含@提及 → 额外推送 → PushService.sendPushMsg() → RocketMQ PUSH_TOPIC
9
最终 WS 推送
PushConsumer (BROADCAST) → WebSocketService.sendToUid() → TextWebSocketFrame(JSON type=4) → 客户端更新 UI

2.6 权限解析流程 (RBAC) Service

1
查询用户所有角色
PermissionService.checkPermission(serverId, userId, channelId, permissionBit) → SQL: role JOIN member_role JOIN server_member WHERE user_id & server_id
2
计算基础权限
effectivePermissions = role1.bits | role2.bits | ... (位 OR 运算)
3
ADMINISTRATOR 短路
if (effectivePermissions & 0x0008) != 0 → 直接返回 true
4
查询频道权限覆盖
SELECT * FROM channel_permission WHERE channel_id (target_type=0 → 角色级 / target_type=1 → 用户级)
5
应用覆盖规则
effectivePermissions = (ep | allow_bits) & ~deny_bits — 优先级: @everyone < 自定义角色 < 角色覆盖 < 用户覆盖
6
检查目标权限位
return (effectivePermissions & targetBit) != 0

2.11 WebSocket 状态机（前端视角）

1
[未连接] → [已连接]
ws://host:8091/ → Netty 30s IdleStateHandler 心跳检测
2
[已连接] → [已认证]
收到 {"type":3} (LOGIN_SUCCESS) — 绑定 uid, avatar, token
3
[已认证] → [订阅中]
进入服务器 → GET /api/v1/servers/{id} → 发 {"type":10} 批量订阅 → 收 {"type":11} ACK
4
在频道中交互
收 type=4(MESSAGE) / type=16(REACTION) / type=17(TYPING) | 发 type=13(TYPING_START) / type=14(TYPING_STOP)
5
Thread 模式
发 {"type":18} (SUBSCRIBE_THREAD) / 发 {"type":19} (UNSUBSCRIBE_THREAD)
6
[断开]
Netty channelInactive() → 清理所有订阅 → UserOfflineEvent
